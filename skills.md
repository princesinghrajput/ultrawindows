# Skills & Context – Screenshot-Based UI Redesign

## Objective
Redesign an existing webpage based on a provided screenshot into a modern, clean, and fully responsive UI.

The redesign must be suitable for a static, SEO-first marketing website.

---

## Input
- Screenshot of an already built webpage
- Visual reference only (no code)

---

## Output
- Redesigned UI mockup
- Desktop-first layout with mobile and tablet adaptations
- Improved visual hierarchy, spacing, and readability
- Premium, professional aesthetic

---

## Core Skills Required

### UI / UX Redesign
- Visual refactoring of existing layouts
- Improving spacing, alignment, and typography
- Reducing clutter and visual noise
- Creating clear content sections
- Strong responsive design thinking

### Responsive Design
- Mobile-first or adaptive layouts
- Logical content reflow for smaller screens
- Touch-friendly spacing and CTAs
- Avoid horizontal scrolling or cramped layouts

### SEO-Aware Layout
- Page-based layout (not app-style)
- One clear H1 per page
- Logical heading hierarchy (H1 → H2 → H3)
- Text content visible by default (not hidden behind tabs)
- Clear page intent above the fold

### Content Preservation & Improvement
- Preserve original intent and meaning of content
- Rephrase only if it improves clarity
- Avoid removing meaningful textual content
- Ensure key content remains crawlable

### Image & Media Handling
- Image placement that supports content
- No text embedded inside images
- Space for captions and alt-context

---

## Constraints
- No JavaScript-driven UI dependencies
- Must be convertible to a static Next.js site later
- SEO and readability take priority over visual effects

---

## Quality Bar
- Clean, modern, production-ready mockup
- Responsive by design
- SEO-safe structure
- Easy to translate into code
